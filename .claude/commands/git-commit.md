Git Commit Assistant

1. **Check Repository Status:**
   - Run `git status` to see current changes
   - Run `git diff` to review specific modifications

2. **Commit Message Guidelines:**
   - Keep description under 50 characters
   - Use lower case and past tense ("added", not "ADD" or "add")
   - Remove any AI model references

3. **Safety Checks:**
   - Verify no sensitive data (API keys, passwords) in changes

4. **Execution:**
   - Stage appropriate files with `git add`
   - Commit with descriptive message
   - Push to appropriate remote branch

5. **Error Handling:**
   - Handle merge conflicts if they arise
   - Check for upstream changes before pushing
   - Provide clear error messages if operations fail
